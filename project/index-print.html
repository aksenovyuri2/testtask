Навигация и «две корзины»
Два сценария в одном приложении с разными корзинами. Market «ниже сгиба» → ложные заходы, дропы на финале при смешанных корзинах.

Лишний клик до ATC
На PLP нет быстрого «+» — даже безатрибутный SKU требует открыть PDP. Импульс гаснет, повтор страдает.
→ ATC CVR · ATC latency
ранг 3
Смешанные корзины
Проблема не в платежах, а в структуре: разные источники → разные правила оформления. Без «моста» — потеря контекста.
→ Checkout CVR mixed
ранг 4

Задержки, недовложения, возвраты в Маркете → разрыв MBC → MDC, падение NPS и повтора.
→ Fulfillment · Repeat

05 · Эксперименты
КАК ИЗМЕРЯЕМ ЭФФЕКТ.
Принципы дизайна
  • Рандомизация — по user_id (стабильный assignment).
  • Статистика — α = 5%, power = 80%, z-test для пропорций.
  • SRM = 0 — обязательный гейт перед чтением результатов.
  • Мощность — n ≈ 16·p̄(1−p̄) / MDE² на группу.
  • Устойчивость — CUPED (если есть ковариаты); diff-in-diff для кластеров.
  • Деноминатор NSM — OMNI MAU; абсолюты — только через aMAU.
Guardrails — обязательны на каждом тесте
  • NPS / CSAT — не хуже базы
  • CM2 / order ≥ 0 (Delivery и C&C)
  • On-time / Cancel / Returns — в нормах
  • Доля отрицательных UCM — не растёт
  • Каннибализация офлайна — geo-holdouts ≤ K%
  • Crash-free / TTI — не хуже
Атрибуция к NSM
ΔMBC = ΔExposure × CTR × Intent
         × ATC × Checkout + …

MDC = MBC × Fulfillment
Definition of Ready
  • События в проде — единые order_id / user_id / flow_id
  • aMAU по сегментам · Fulfillment_success по регионам
  • AOV_cat и share_cat (Delivery / C&C)
  • Дашборд: NSM (1/7/28), 6 шагов воронки, guardrails, очередь A/B
  • Runbook: критерии остановки, rollback, коммуникация
Circuit breakers
CM2<0 → авто-кап инсентивов / порог free / слоты On-time<Y% → throttle слотов + честный ETA Drop@Mixed↑ → «мост» принудительно SRM≠0 → rollback, re-seed

КАКИЕ РУЧКИ ДВИГАЮТ P&L.
CM2 / order — на заказ
СплитЧто разрезаем
КаналDelivery vs C&C
Корзина< / ≥ порога free
ОплатаКарта · СБП · Мир Pay
Регион / СЦCapability, расстояния
КатегорииFMCG · GM · Питомцы · Красота · Хоз
Break-even free-delivery
AOV* ≥ (L + i) / g%
L — переменная логистика. i — инсентивы. g — валовая маржа. Авто: при росте L (погода / пик) → AOV* ↑ или слоты сужаются.
Панель рычагов CPO
Delivery / слоты
  • Capacity-aware слоты
  • AOV-порог free — динамический
  • «Тяжёлые» — Delivery с явным fee
Оплата
  • СБП / Мир Pay на повторах
  • Token reuse в 1-тап
ATC / AOV
  • Автодобор до AOV*
  • Бандлы миссий
  • «Мои 20 / Покупай снова»
Fulfillment
  • 1-клик претензия с типизацией
  • SLA саппорта
  • ↓ ExpReturns → ↑ CM2
Авто-правила A. CM2 / order < 0 в Delivery → AOV* +Δ, скрыть «минусовые» SKU, push к C&C. B. Sₙ% по категории K → выключить SKU-промо, бандлить с «плюсовыми». C. On-time < Y% в СЦ → сузить слоты 24 ч, честный ETA, пересчёт порога free.

ЧЕК-ЛИСТ ДАННЫХ.
1 · Идентичность
  • Сквозные ключи user_id · device_id · session_id · flow_id · order_id
  • source = omni_inapp | omni_deeplink | market_standalone
  • Анти-дубликаты: идемпотентность по (order_id, payment_id, ts)
  • NSM «месяц» = календарный, UTC + local offset для витрин
  • aMAU-флаги: availability, eligibility, exposure_seen
2 · События
Воронка
view_shop_entry · click_shop_entry · tab_source_select · plp_quick_add · plp_variant_open/select · multiadd_select/submit · cart_mixed_flag · cta_split_checkout_click · checkout_flow_id · pay_success/fail
Post-purchase
order_status_update{status, eta, band} · eta_risk_flag · claim_one_click_open/submit · claim_auto_resolved · contact_reason
СЛА: лаг ≤ 5 мин · дроп пакетов ≤ 0.3%
3 · Справочники
  • Каталог: sku_id, категория, атрибутность, вес/объём/темп.
  • Продавцы / магазины / СЦ, лог-зоны, правила слотов
  • Платежи: методы + фактические комиссии
  • OMS / ETA: статусы, SLA по СЦ
  • AOV_cat / share_cat — отдельно Delivery vs C&C
4 · Метрики · NSM = MBC
MBC = uniq buyers (omni in-app / deeplink) / OMNI MAU
MDC = MBC × Fulfillment_success
Исключаем: standalone, тест / фрод, отмены до исполнения.
5 · Quality gates до A/B
  • A/A 48 ч → SRM = 0 · расхождение ±0.5 п.п.
  • Монотонность воронки: Views ≥ CTR ≥ Intent ≥ ATC ≥ Checkout
  • Пропуски ключей < 0.5% · дубли < 0.5%
  • Согласованность ID сквозная ≥ 99.7%
  • В числителе NSM нет standalone (по source)
